# 赛博吸氧机 / Cyber Copium - 项目功能与能力边界

## 产品概述

**产品名称**: 赛博吸氧机 / Cyber Copium  
**产品类型**: 面向年轻用户的 AI 情绪转译与微行动工具  
**产品形态**: Web 单页应用  
**一句话定位**: 当现实太窒息时，给用户一口"可转发、可执行、带一点荒诞感"的赛博精神氧气。

---

## 已实现功能

### 1. 输入模块 ✅
- **文本输入框**: 支持 200 字以内的困境描述输入
- **示例语句快速填充**: 提供预设示例，点击即可填充输入框
- **输入验证**: 非空检查、长度限制
- **加载状态**: 提交后显示加载动画
- **手绘风格输入框**: 带轻微旋转的手绘风格设计

### 2. AI 生成模块 ✅
- **阿里云百炼 API 集成**: 使用 qwen-flash 模型，兼容 OpenAI 格式
- **Prompt v1 模板**: 输出五件套结构化内容
- **状态识别**: 5 种状态标签（拖延/回避、羞耻/自我否定、疲惫/低启动、轻度发疯/情绪过载、想做但卡住）
- **语气强度控制**: L1/L2/L3 三挡模式
- **错误兜底机制**: API 失败时返回 fallback 数据
- **贴纸文案长度约束**: ≤15 汉字（硬性约束）

### 3. 结果展示模块 ✅
- **五件套卡片展示**:
  - 🏷️ 状态命名 (≤20 字)
  - 👄 抽象嘴替 (≤50 字)
  - 🎭 荒诞动机 (≤40 字)
  - ⚡ 最小行动 (≤30 字，3 分钟内可执行)
- **精神贴纸预览**: 带角落标记的贴纸卡片
- **反馈按钮**: 重新生成、我去做了、点赞反馈

### 4. 手绘风格设计 ✅
- **SVG 手绘滤镜效果**: 使用 feTurbulence 和 feDisplacementMap
- **纸张纹理背景**: 米白色 (#fffef7) + SVG noise 图案
- **硬阴影卡片**: 无模糊的实色阴影效果
- **轻微旋转动画**: 卡片带轻微旋转变换
- **手绘装饰元素**: ✦ ～ ✎ 等符号装饰
- **手写字体**: Caveat (标题) + Kalam (正文)

### 5. 技术实现 ✅
- **前端**: Next.js 14 + TypeScript + Tailwind CSS
- **后端**: Next.js Route Handlers (/api/generate)
- **UI 组件**: 自定义手绘风格组件
- **动画**: CSS 过渡动画
- **测试**: Playwright 前端自动化测试
- **部署**: Vercel (配置中)

## 数据结构

### GenerationResult 接口
```typescript
interface GenerationResult {
  status_label: string;      // 状态命名 (≤20 字)
  voice_line: string;        // 抽象嘴替 (≤50 字)
  sticker_text: string;      // 贴纸文案 (≤15 字) ⚠️ 硬性约束
  absurd_motivation: string; // 荒诞动机 (≤40 字)
  micro_action: string;      // 最小行动 (≤30 字)
  tone_level: 'L1' | 'L2' | 'L3';
  status_tag: StatusTag;
}
```

### 状态标签
- 拖延/回避
- 羞耻/自我否定
- 疲惫/低启动
- 轻度发疯/情绪过载
- 想做但卡住

### 语气强度
- **L1 稳定模式**: 低梗、低刺激、偏现实支持（适合高风险场景）
- **L2 轻抽象模式**: 轻微自嘲、温和荒诞（默认模式）
- **L3 高抽象模式**: 允许更明显的发疯文学（用户明确想发疯时）

## 能力边界

### 项目范围边界

**✅ 在范围内**:
- 单页 Web 应用，无复杂路由
- 基于用户输入生成五件套内容
- 贴纸预览和保存
- 基础反馈功能（点赞、重新生成、我去做了）
- 手绘涂鸦风格 UI
- 纯黑白配色方案

**❌ 不在范围内** (MVP 阶段不做):
- 社区 / 评论 / 关注体系
- 长周期陪伴与人格养成
- 多轮复杂对话
- 高自由度 AI 图像生成
- 复杂账号体系
- 历史记录功能
- 心理治疗与诊断相关功能

### 技术边界

**前端边界**:
- 使用 Next.js App Router
- 客户端组件使用 'use client' 指令
- localStorage 存储用户输入（临时）
- 无服务端状态管理

**后端边界**:
- 单 API 路由 (/api/generate)
- 无数据库持久化（MVP 阶段）
- 无用户认证
- 环境变量配置 API Key

**AI 边界**:
- 单次生成，非对话式
- 基于阿里云百炼 API
- Prompt 模板固定，无动态调整
- 无语料库检索（MVP 阶段）

### 内容边界

**输出约束**:
- 贴纸文案 ≤15 汉字（硬性约束）
- 状态命名 ≤20 字
- 抽象嘴替 ≤50 字
- 荒诞动机 ≤40 字
- 最小行动 ≤30 字

**安全边界**:
- 高风险输入降级为稳定表达
- 不输出伤害性、羞辱性内容
- 不做诊断、治疗、疗效承诺
- 抽象强度由状态识别控制

## 核心价值

1. **语境命中**: 让用户感觉"这话像我会说的"
2. **情绪转译**: 把混乱情绪翻译成可观看、可转发的内容
3. **行动收束**: 通过"荒诞动机"平滑过渡到 3 分钟内可执行的动作
4. **风格可控**: 根据场景动态控制抽象强度

---

## 典型使用场景

### 场景一：求职卡住
- **输入**: "我又不想投简历了。"
- **输出**: 状态命名 + 嘴替文案 + 贴纸 + 荒诞动机 + "只改简历标题"的微动作

### 场景二：学习摆住
- **输入**: "今天一天都没学进去。"
- **输出**: 将"我废了"转译为"当前处于高阻力低启动状态"

### 场景三：想发疯
- **输入**: "我现在只想发疯。"
- **输出**: 允许情绪表达，控制抽象强度，给出不刺激的小动作
