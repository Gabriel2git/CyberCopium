# PostHog 有事件但 Insight 没数据排查计划

## 问题描述

- PostHog Events 中可以看到事件
- 但 Insight 看板没有数据显示
- 这是一个具体的配置问题

## 可能原因

### 1. 时间范围设置问题
- Insight 默认时间范围可能与数据时间不匹配
- 需要检查时间筛选器

### 2. Event 名称大小写问题
- PostHog 事件名称大小写敏感
- Events 页面显示的名称可能与代码中发送的不一致

### 3. 筛选器/过滤条件问题
- Insight 可能设置了错误的筛选条件
- 导致数据被过滤掉

### 4. Breakdowns 问题
- 如果使用了 breakdown，空白值不会显示

### 5. Event Properties 问题
- 某些属性可能是 undefined 或 null

## 排查步骤

### 步骤 1：确认事件名称
1. 打开 Events 页面
2. 找到你看到的事件名称
3. 复制完整的事件名称（包括大小写）

### 步骤 2：创建最简单的 Insight
1. 创建新的 Insight
2. 只选择一个事件（不加任何筛选）
3. 不要添加任何 breakdown
4. 选择 "Last 7 days" 时间范围
5. 查看是否有数据

### 步骤 3：检查时间范围
- 确认时间范围包含今天的数据
- 尝试选择 "Today" 或 "Last 24 hours"

### 步骤 4：检查筛选器
- 移除所有筛选条件
- 只保留事件名称

### 步骤 5：对比 Events 和 Insight
- 确认两边使用的事件名称完全一致

- [ ] 移除所有 breakdown

## 常见解决方案

### 方案 1：重新创建 Insight
有时 PostHog 的 Insight 会缓存错误配置。尝试：
1. 删除旧的 Insight
2. 重新创建一个新的
3. 只添加事件名称，其他都不加

### 方案 2：检查事件名称
Events 页面：
```
page_view
```
但代码中可能是：
```
Page View  (注意大小写)
```

### 方案 3：刷新 PostHog 缓存
有时需要等待几分钟，让 PostHog 处理新的事件数据
