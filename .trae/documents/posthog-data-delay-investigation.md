# PostHog 数据延迟问题排查计划

## 问题描述

- PostHog 看板只有 3月9日 的数据
- 今天（3月10日）使用过应用，但看板没有数据
- 数据是否有延迟？

## 可能原因分析

### 1. PostHog 数据处理延迟
- **免费版**：数据延迟 5-15 分钟
- **付费版**：实时数据

### 2. Session Replay 延迟
- Session Replay 数据可能需要更长时间处理
- 通常延迟 10-30 分钟

### 3. 本地开发 vs 生产环境
- 本地开发环境的数据可能不会发送到 PostHog
- 需要检查 `.env.local` 配置

### 4. 广告拦截器
- 之前遇到过广告拦截器阻止 PostHog 脚本加载

### 5. PostHog 配置问题
- 项目 API Key 是否正确
- 是否正确初始化

## 排查步骤

### 步骤 1：检查浏览器控制台
在应用页面打开浏览器控制台，检查：
1. PostHog 是否成功初始化
2. 是否有事件发送
3. 是否有错误信息

### 步骤 2：检查 PostHog Live Events
- 路径：`Activity` → `Live`
- 查看是否有实时事件流入

### 步骤 3：检查网络请求
- 在 Network 面板过滤 `posthog`
- 查看是否有请求发送到 PostHog

### 步骤 4：检查环境配置
确认 `.env.local` 中的配置：
```
NEXT_PUBLIC_POSTHOG_KEY=phc_xxx
NEXT_PUBLIC_POSTHOG_HOST=https://us.i.posthog.com
```

### 步骤 5：测试手动发送事件
在应用页面点击测试按钮，验证事件是否能发送

## 常见解决方案

### 方案 1：等待数据处理
- PostHog 免费版数据延迟 5-15 分钟
- 建议等待 15 分钟后刷新

### 方案 2：检查广告拦截器
- 禁用广告拦截插件
- 刷新页面重试

### 方案 3：检查 PostHog 项目
- 确认使用的是正确的 Project
- 确认 API Key 匹配

### 方案 4：检查本地开发环境数据
- 本地开发可能不发送数据到 PostHog
- 需要在生产环境测试

## 验证检查清单

- [ ] 浏览器控制台无 PostHog 错误
- [ ] PostHog 初始化成功
- [ ] 手动测试事件可以发送
- [ ] Network 面板有发往 posthog.com 的请求
- [ ] PostHog Live Events 有数据流入
- [ ] 等待 15 分钟后刷新看板

## 今日数据为 0 的可能原因

1. **广告拦截器**：仍有可能阻止部分请求
2. **项目配置错误**：使用了错误的 API Key
3. **本地开发数据**：本地环境没有正确发送数据
4. **浏览器缓存**：需要清除缓存后重试

## 建议操作

1. 先等待 15 分钟，数据可能正在处理
2. 打开浏览器控制台，查看是否有错误
3. 在 PostHog 中查看 Live Events
4. 如果还是不行，检查环境配置
