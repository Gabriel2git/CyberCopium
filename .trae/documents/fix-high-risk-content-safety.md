# 修复高危情绪内容安全问题

## 问题分析

当用户输入高危情绪（如"我想死了"）时，系统虽然正确识别为：
- `risk_level: high`
- `tone_level: L1`（强制降级）

但生成的文案仍然包含刺激性内容：
- **抽象嘴替**: "世界他妈的太吵了，但你还没死，说明它还留着你当备用电池。"
- **荒诞动机**: "你要是真死了，地球还得专门给你办个追悼会，那多麻烦。"

这些内容明显不符合 L1 "温和兜底、多给安全感" 的要求，反而可能加剧用户的负面情绪。

## 问题根源

当前的 `COMPOSER_PROMPT` 存在以下问题：

1. **L1 定义不够具体** - 只说"少玩梗，多给安全感"，但没有明确禁止什么
2. **缺乏高危场景特殊处理** - 没有针对 `risk_level: high` 的特殊约束
3. **示例不够温和** - 即使 L1 模式下，示例仍然带有"备用电池"、"追悼会"等刺激性隐喻

## 修复方案

### 方案 A: 强化 L1 约束（推荐）

在 `COMPOSER_PROMPT` 中增加：

```
## 3. 语气约束（关键）
根据 {tone_level} 和 {risk_level} 控制内容：

### L1 + risk_level=high（高危情绪，必须严格遵守）
- 🚫 严禁使用：死亡、自杀、结束生命、去死、死了一了百了等相关词汇
- 🚫 严禁使用：他妈的、备用电池、追悼会、葬礼、墓地等刺激性隐喻
- 🚫 严禁暗示："你还没死"、"死了会麻烦"等暗示死亡可能性的表述
- ✅ 必须使用：温暖、陪伴、暂时、会过去、有人在乎、值得被爱等词汇
- ✅ 必须传递："这种痛苦是暂时的"、"你值得被温柔对待"、"有人在乎你"的信息
- ✅ 语气要求：像一位温柔的朋友在深夜陪伴，不评判、不说教、只给予接纳

### L1 + risk_level=low/medium（一般低电量场景）
- 少玩梗，多给安全感
- 避免刺激性表达

### L2（默认模式）
- 适度自嘲，温和荒诞

### L3（高抽象模式）
- 允许发疯文学
```

### 方案 B: 高危场景专用 Prompt

当检测到 `risk_level: high` 时，使用完全不同的 Prompt：

```typescript
const HIGH_RISK_COMPOSER_PROMPT = `...更温和的 Prompt...`;

// 在 API 路由中
if (analysis.risk_level === 'high') {
  systemPrompt = HIGH_RISK_COMPOSER_PROMPT;
} else {
  systemPrompt = COMPOSER_PROMPT;
}
```

在生成结果后，增加一层安全检查：


## 我的建议

推荐 **方案 A + 方案 C** 结合：

1. **方案 A** - 在 Prompt 层面明确约束，从源头避免生成刺激性内容
2. **方案 C** - 作为兜底保障，即使 Prompt 失效也能拦截危险内容
