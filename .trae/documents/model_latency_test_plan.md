# 模型延迟测试计划

## 目标
测试三个 Qwen 模型的 API 响应延迟，选择最快的模型用于生产环境。

## 待测试模型
1. `qwen3.5-flash-2026-02-23` - 最新版本
2. `qwen3.5-flash` - 标准版本
3. `qwen-flash` - 当前使用的版本

## 测试方法

### 测试脚本设计
创建一个测试脚本，对每个模型进行以下测试：

1. **单次调用测试**
   - 使用相同的 prompt
   - 记录首次响应时间（TTFB - Time To First Byte）
   - 记录完整响应时间

2. **多次调用测试**
   - 每个模型调用 5 次
   - 计算平均延迟
   - 计算中位数延迟
   - 记录最大/最小延迟

3. **并发测试**（可选）
   - 同时发起多个请求
   - 测试模型在高并发下的表现

### 测试环境
- 使用阿里云百炼 API
- 相同的网络环境
- 相同的测试时间（避免高峰期）

## 实施步骤

### Phase 1: 创建测试脚本
1. 创建 `scripts/test-model-latency.ts`
2. 实现延迟测试逻辑
3. 添加结果输出和统计

### Phase 2: 执行测试
1. 运行测试脚本
2. 收集每个模型的延迟数据
3. 生成对比报告

### Phase 3: 结果分析
1. 比较三个模型的平均延迟
2. 分析延迟稳定性（方差/标准差）
3. 确定最优模型

### Phase 4: 更新生产配置
1. 修改 `src/app/api/generate/route.ts`
2. 将模型名称更新为延迟最低的模型
3. 测试生产环境
