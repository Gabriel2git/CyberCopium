# 压缩大模型延迟优化计划

## 目标
通过多维度优化，显著降低 API 响应延迟，提升用户体验。

## 当前问题分析

### 现状
- 模型本身延迟：1.85 秒（qwen-flash，已是最快）
- 实际用户体验：可能感觉更慢
- 架构：双层调用（Analyzer + Composer）

### 延迟来源分析
1. **网络传输** - 用户→Vercel→阿里云
2. **Vercel Cold Start** - Serverless 冷启动时间
3. **双层 API 调用** - Analyzer + Composer 两次请求
4. **Prompt 长度** - 长 Prompt 增加处理时间
5. **模型推理** - 1.85 秒（已优化）

## 优化策略

### 策略 1: 调整 Prompt（影响中等）

#### 1.1 压缩 Prompt 长度
- **当前**: ANALYZER_PROMPT + COMPOSER_PROMPT 都很长
- **优化**: 
  - 删除冗余说明
  - 使用更简洁的表述
  - 移除不必要的示例
- **预期收益**: 减少 10-20% 延迟

#### 1.2 优化 max_tokens
- **当前**: Analyzer 300, Composer 500
- **优化**:
  - Analyzer: 300 → 200（分析结果通常很短）
  - Composer: 500 → 350（减少不必要的生成）
- **预期收益**: 减少 15-25% 延迟

#### 1.3 提高 temperature
- **当前**: Analyzer 0.3, Composer 0.7
- **优化**: 适当提高 temperature 可能减少推理时间
- **风险**: 可能影响输出质量，需要测试

### 策略 2: 架构优化（影响大）

#### 2.1 单层架构替代双层
- **当前**: Analyzer → Composer 两次调用
- **优化**: 合并为单次调用
- **实现方式**:
  - 创建一个统一的 Prompt
  - 同时完成分析和生成
- **预期收益**: 减少 50% 延迟（去掉一次 API 调用）
- **风险**: 可能降低生成质量

#### 2.2 流式响应 (Streaming)
- **当前**: 等待完整响应后返回
- **优化**: 使用流式输出，用户先看到部分内容
- **预期收益**: 感知延迟大幅降低
- **实现复杂度**: 中等

#### 2.3 预生成 + 缓存
- **当前**: 每次请求都实时生成
- **优化**: 缓存常见输入的结果
- **实现方式**:
  - 对常见场景预生成内容
  - 使用 Redis/Vercel KV 缓存
- **预期收益**: 缓存命中时延迟 <100ms

### 策略 3: 基础设施优化（影响大）

#### 3.1 解决 Vercel Cold Start
- **问题**: Serverless 函数冷启动可能增加 1-3 秒
- **解决方案**:
  - 使用 Vercel Pro 的 "Warm" 功能
  - 配置 Cron Job 定期预热
  - 迁移到常驻服务器（如 Railway/Render）
- **预期收益**: 减少 1-3 秒冷启动时间

#### 3.2 Edge Function 优化
- **当前**: Serverless Function
- **优化**: 使用 Edge Function 减少网络跳转
- **预期收益**: 减少 100-300ms 网络延迟

#### 3.3 就近部署
- **当前**: Vercel 默认节点
- **优化**: 配置阿里云函数计算（国内）
- **预期收益**: 减少 200-500ms 网络延迟

### 策略 4: 用户体验优化（感知优化）

#### 4.1 优化 Loading 状态
- **当前**: 简单的 loading 动画
- **优化**:
  - 添加进度条或步骤指示
  - 显示 "正在分析..." → "正在生成..."
  - 添加有趣的等待文案
- **预期收益**: 感知等待时间减少 30%

#### 4.2 骨架屏 (Skeleton Screen)
- **优化**: 在内容生成前显示骨架屏
- **预期收益**: 感知延迟降低

#### 4.3 渐进式加载
- **优化**: 先显示分析结果，再显示完整内容
- **预期收益**: 用户先看到部分内容，感知更快

### 策略 5: 监控与诊断

#### 5.1 添加详细日志
- 记录每个阶段的耗时
- 识别真正的瓶颈

#### 5.2 性能监控
- 集成 Vercel Analytics
- 监控 API 响应时间
- 设置告警阈值
