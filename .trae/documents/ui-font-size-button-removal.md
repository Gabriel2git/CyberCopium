# UI 优化计划 - 字体调整和删除切换按钮

## 问题描述
用户提出了三个 UI 优化需求：
1. 标题"赛博吸氧机 / Cyber Copium"需要换成手写字体
2. "试试这些"示例按钮和 Footer 警告文字需要放大
3. 删除右上角的主题切换按钮

## 当前状态分析
- 标题"赛博吸氧机"已使用 Kalam 手写字体（在 Hero 组件中）
- "Cyber Copium"已使用 Kalam 手写字体
- 示例按钮文字使用的是 Caveat 字体，但可能需要放大
- Footer 文字使用的是 Caveat 字体，但可能需要放大
- 右上角切换按钮（ThemeToggle）可能还在某些地方使用

## 实施步骤

### 步骤 1：检查并确认标题字体
- 检查 Hero 组件中"赛博吸氧机"和"Cyber Copium"的字体
- 确认已使用 Kalam 手写字体
- 如需调整，确保中文字体也能保持手写感

### 步骤 2：放大示例按钮文字
- 定位：InputSection 组件中的示例按钮
- 当前：`text-sm px-4 py-2`
- 修改为：`text-base px-4 py-2` 或更大
- 同时放大"试试这些："标签

### 步骤 3：放大 Footer 警告文字
- 定位：Footer 组件
- 当前：`text-sm`
- 修改为：`text-base` 或 `text-lg`
- 保持 Caveat 手写字体

### 步骤 4：删除右上角切换按钮
- 检查所有页面是否还在使用 ThemeToggle 组件
- 从首页移除（如果存在）
- 从结果页移除（如果存在）
- 确认完全删除

### 步骤 5：验证和测试
- 检查首页字体显示效果
- 检查示例按钮大小
- 检查 Footer 文字大小
- 确认无切换按钮
- 验证整体视觉效果

## 验收标准
- [ ] 标题使用手写字体（Kalam）
- [ ] 示例按钮文字明显放大
- [ ] Footer 警告文字明显放大
- [ ] 右上角无切换按钮
- [ ] 整体视觉协调统一
- [ ] 所有字体保持手绘风格
