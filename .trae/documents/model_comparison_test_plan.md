# 模型速度与质量对比测试计划

## 目标

测试 4 个模型的：
1. **响应速度** - 筛选出 5 秒内响应的模型
2. **回答质量** - 评估输出质量
3. **超时率** - 超过 5 秒的比例

## 测试模型

| 模型 | 说明 |
|------|------|
| qwen-flash | 当前使用，最快 |
| qwen-plus | 升级版，更强 |
| qwen3.5-plus | 3.5 版增强 |
| qwen3.5-flash | 3.5 版快速（之前测试很慢） |

## 测试配置

### 超时设置
```typescript
timeout: 5000  // 5 秒超时
```

### 质量评估标准

| 指标 | 评分标准 |
|------|----------|
| JSON 格式正确 | 能正确解析为 JSON |
| 字段完整 | 包含所有必要字段 |
| 内容相关性 | 输出与输入相关 |
| 风格一致性 | 符合指定的风格模式 |
| 字数合规 | 各字段在限制字数内 |

### 测试输入（5 个典型场景）

1. **职场压力**：`最近工作压力好大，每天都想辞职但又不敢，感觉自己快崩溃了`
2. **学业拖延**：`明天要交论文了，但我一直拖延，完全不想动怎么办`
3. **人际关系**：`和男朋友吵架了，感觉很委屈，不想说话`
4. **日常焦虑**：`每天晚上都睡不着觉，想太多，感觉生活没有意义`
5. **情绪崩溃**：`今天被老板骂了，觉得自己特别失败，什么都做不好`

## 测试方法

### 1. 速度测试

每个模型对每个输入测试 3 次，记录：
- 响应时间
- 是否超时
- 是否成功返回

### 2. 质量测试

对成功返回的结果进行质量评分：
- JSON 格式：10 分
- 字段完整：20 分
- 内容相关：30 分
- 风格一致：20 分
- 字数合规：20 分

总分：100 分

### 3. 综合评估

```
综合得分 = 速度得分 × 50% + 质量得分 × 50%

速度得分 = (成功次数 / 总测试次数) × 100
```
